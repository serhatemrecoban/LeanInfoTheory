Roadmap

A 3 / 6 / 12 month plan for the library and certificate checker.

This roadmap separates completed Lean/site work from active polish, planned theorem work, and later infrastructure milestones. The labels are intentionally conservative: they describe the current repository, not aspirations.

Status Snapshot

Status Current meaning Examples
Completed Checked Lean work or completed website/documentation work. Finite PMF information measures, semantic/KL bridge theorems, submodularity certificate demo, generated reference artifacts, and website steps 1-6.
Active Current near-term maintenance or presentation work. Keeping theorem highlights, module guide, roadmap, and demo pages aligned with the Lean code.
Planned Next technical goals that are not yet formalized or generated. More textbook inequalities, richer certificate assumptions, theorem-level blueprint, blueprint PDF, and full Lean doc-gen output.
Later Larger infrastructure or research-facing milestones. PSITIP/oXitip import, network converse skeletons, and a technical report or paper.

Completed

  • Completed Finite PMF-based entropy, conditional entropy, mutual information, and conditional mutual information.
  • Completed Entropy relabeling, coordinate swap, reassociation, upper-bound, and uniform-law equality theorems.
  • Completed Semantic bridge theorems for expected self-information, conditional laws, KL divergence, nonnegativity, and a first chain rule.
  • Completed demo Submodularity from a Lean-validated conditional-mutual-information certificate.
  • Completed Website dashboard, theorem highlights, module guide, source-derived declaration index, module dependency map, prior-art page, and trusted-flow demo diagram.

Active

  • Active Keep public website pages aligned with the current Lean code and terminology.
  • Active Maintain project notes in the foundation conventions, project log, and website-task note.
  • Active Choose the next finite-information-theory theorem milestone after the completed semantic bridge plan.

Planned

  • Planned Extend the current module dependency map and source-derived declaration index toward theorem-level leanblueprint pages, a blueprint PDF, and full Lean doc-gen output.
  • Planned Add more checked textbook entropy inequalities beyond the current submodularity demo.
  • Planned Add richer certificate assumptions such as independence, functional dependence, and Markov constraints.
  • Planned Develop more of the Cover-Thomas Chapter 2 theorem layer, such as data processing, Fano, and broader chain-rule APIs.
  • Planned Prepare focused mathlib PRs for generic, stable finite-measure lemmas.

Later

  • Later Add PSITIP/oXitip-style import into untrusted raw certificate data.
  • Later Formalize selected network-information-theory converse proof skeletons.
  • Later Write a technical report or short paper describing the Lean library and certificate pipeline.
