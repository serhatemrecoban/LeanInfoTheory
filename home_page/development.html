Build

git clone https://github.com/serhatemrecoban/LeanInfoTheory.git
cd LeanInfoTheory
lake exe cache get
lake build LeanInfoTheory

The repository currently uses Lean v4.30.0 and mathlib v4.30.0.

Focused Checks

lake build LeanInfoTheory.Shannon.SemanticBridge
lake build LeanInfoTheory.Certificate.Submodularity
lake build LeanInfoTheory.Examples

Website Generation

python scripts/generate_website_blueprint.py
python scripts/generate_website_api_index.py
python scripts/check_website.py

Run these after changing Lean imports, public declarations, or doc comments. They regenerate the website's module-level dependency map, generated declaration index, and machine-readable graph/index data. The checker validates generated JSON and local website links. CI checks that the generated files are current.

Project Hygiene

  • The CI build includes a strict placeholder scan for Lean source files.
  • Core finite Shannon files keep imports light.
  • Heavier KL, conditional-law, and semantic theorem imports live in the semantic bridge layer.
  • Certificate examples are separately importable so the base finite API stays small.

Import Guide

  • Use LeanInfoTheory.Shannon.Entropy for finite entropy basics.
  • Use LeanInfoTheory.Shannon.InfoMeasures for marginals, conditional entropy, mutual information, and conditional mutual information.
  • Use LeanInfoTheory.Shannon.EntropyBounds when the Jensen-based upper bound or uniform equality theorem is needed.
  • Use LeanInfoTheory.Shannon.SemanticBridge for expected self-information, conditional-law, KL, nonnegativity, and chain-rule theorems.
  • Use LeanInfoTheory.Certificate.Submodularity for the first certificate-backed inequality demo.
