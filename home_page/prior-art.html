mathlib

The project is intentionally mathlib-based. It reuses mathlib PMFs, finite types, real analysis, products of measures, Markov kernels, PMF.toMeasure, binary and q-ary entropy, KL divergence, and coding-theory anchors such as Kraft-McMillan.

PFR Formalization

The Lean PFR project is an important reference for entropy and mutual information in a serious formalization. LeanInfoTheory keeps the first project layer finite and PMF-oriented, while auditing PFR and mathlib APIs before adding more general measure-theoretic layers.

Rocq Infotheo

Rocq's Infotheo library is a useful comparison point for finite distributions, conditional distributions, and textbook-style information measures. LeanInfoTheory follows Lean and mathlib naming and typeclass conventions first, but uses Infotheo as a sanity check for whether definitions can scale to larger theorem families.

PSITIP and oXitip

PSITIP and oXitip are relevant for entropy-inequality certificate search. The planned architecture is that external tools may search for certificates and export raw data, but Lean remains the trusted checker that validates the certificate and proves the target inequality.

Textbook Anchors

The near-term mathematical path is closest to the finite discrete foundations in Cover and Thomas, El Gamal and Kim, Yeung, and Polyanskiy and Wu. The project does not try to mirror a textbook line by line; it follows mathlib-compatible definitions and then records which textbook statements have been formalized.

What This Project Adds

LeanInfoTheory currently adds a finite PMF-based Shannon layer, semantic bridges to expected self-information and KL divergence, and a Lean-checked certificate pipeline for primitive Shannon inequalities.
